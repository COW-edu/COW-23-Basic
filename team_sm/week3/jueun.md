## CSS:

    - Cascading(계단식), Style(멋을 내다), Sheets(종이 한 장) --> 태그가 태그를 포함하는 계단식으로 HTML 문서에 스타일을 더해줌.
    - 웹 문서의 콘텐츠에 스타일을 추가해주는 언어이므로 반드시 HTML 문서 작성이 선행되어야 함. CSS 문서 자체는 독립적으로 존재 가능하긴 함.(이땐 단순 텍스트에 불과함)
    - 확장자: *.css

## CSS 기본 문법:

    - 선택자 {속성명: 속성값;}
    - 선택자: 어떤 요소에 스타일을 적용할건지. 나는 이 요소에 스타일을 적용하고 싶다!
    - 중괄호: 선택한 요소에 적용할 스타일을 정의할 영역.
    - 속성명: 어떤 스타일을 정의하고 싶은지(색상, 크기, 두께 등)
    - 속성값: 그 속성을 어떻게 정의하고 싶은지.

## CSS 주석:

    - /*이 안에 작성하면 주석으로 처리되어 사람에겐 보이나 컴퓨터에겐 보이지 않음. 코드에 대한 메모를 남기기 위해 사용*/

## 인라인 스타일:

    - <p style="color: blue;"> 이런 식으로 태그에 style 속성을 추가해 요소에 직접적으로 스타일을 정의함.
    - HTML과 CSS를 분리하기 위해 사용하지 않는게 좋음.

## 스타일 태그:

    - HTML문서에 <style></style> 태그를 추가해 이 안에 CSS 코드 작성 가능.

## link를 이용한 문서 간의 연결:

    - 확장자가 *.css인 파일을 만들어 그 안에 CSS 코드를 작성한 후 HTML 문서에 이를 연결 가능.
    - <link href="index.css" rel="stylesheet">로 사용가능.
    - <link> 태그는 <head></head> 내부에서 사용해야함.
    - href: 연결하고자 하는 외부 소스의 url을 기술하는 속성.
    - rel: 현재 문서와 외부 소스의 연관 관계를 기술하는 속성.

## 전체 선택자(\*):

    - *(애스터리스크)는 '문서 내의 모든 요소'를 의미하는 기호로, 모든 요소를 선택한다.

## 태그 선택자(= 유형 선택자):

    - 주어진 이름을 가진 요소를 선택함. 여러개도 가능.

## 클래스 선택자(.class):

    - 주어진 class 속성값을 가진 요소를 선택함.

## 아이디 선택자(#id):

    - 주어진 id 속성값을 가진 요소를 선택한다.
    - id는 고유한 식별자 역할을 하는 전역 속성이다.

## 그룹 선택자(,):

    - 다양한 유형의 요소를 한꺼번에 선택하고자 할 때 쉼표를 이용해 선택자를 그룹화한다.
    - ex) h1, p, div {color: blue;}

## 선택자 우선순위:

    - 아이디 > 클래스 > 태그
    - 선택자가 겹치는 경우, 나중에 작성된 스타일이 적용됨.

## 텍스트 관련 속성:

    - font-family: 글꼴 정의, 여러개 글꼴 적어서 우선순위 정할 수 있음.
    - font-size: 글자 크기 정의, 수치와 단위를 지정
    - text-align: 왼쪽, 오른쪽, 가운데, 양끝(justify)으로 정렬 방식 정의
    - color: 색 키워드, RGB 색상 코드, RGB함수를 이용해 글자 색상 정의

## 수치:

    - px: 모니터 상의 화소 하나 크기에 대응하는 절대적인 크기
    - rem: <html> 태그의 font-size에 대응하는 상대적인 크기
    - em: 부모태그의 font-size에 대응하는 상대적인 크기

## color:

    - ex) span{ color: red; }
    - ex) span{ color: #FF0000; }
    - ex) span{ color: rgb(100%, 0%, 0%); }

## display 속성:

    - 선택된 요소를 블록과 인라인 요소 중 어느 쪽으로 처리할지 정의함.
    - 속성값: 미리 정의되어 있는 키워드로 지정함.
    - inline: 인라인으로 처리
    - block: 블록 레벨로 처리
    - inline-block: 인라인으로 배치하되, 블록 레벨 요소의 속성을 추가할 수 있도록 처리.
    - none: 디스플레이(표시)하지 않음.

## border 속성:

    - 요소가 차지하고 있는 영역에 테두리를 그릴 수 있음
    - 두께, 모양, 크기를 한 번에 지정 가능한 단축속성.
    ex) span{ border: 1px solid green; }
    - solid(직선 모양)

## border 속성의 하위 속성:

    - border-color: color 정의 방식과 동일
    - border-width: thin, medium, thick 등 키워드 또는 수치 단위.
    - border-style: none(기본값), solid, dotted(점선), dashed(긴 점선) 등

## 박스모델 (Box-Model):

    - 브라우저가 요소를 렌더링 시, 각 요소는 기본적으로 사각형 형태로 영역을 차지함. 이 영역을 '박스'라 표현함.
    - CSS는 박스의 크기, 위치, 속성(색, 배경, 테두리 모양 등)을 결정할 수 있음.
    - 하나의 박스를 구성하는 네 개의 영역: 콘텐츠 영역, 안쪽 여백, 경계선, 바깥쪽 여백.

## 박스 각 영역의 크기 조절하는 방법:

    - 콘텐츠 영역: width, height
    - 콘텐츠와 테두리 안쪽 여백: padding
    - 경계선: border-width
    - 콘텐츠와 테두리 바깥쪽 여백: margin
    - 각 면에 개별적으로 두께를 정의할 수 있음.
    ex) padding-top, margin-right, padding-bottom

## 여러 값을 한 번에 정의하기

    - padding과 margin은 네 면의 여백에 대한 '단축속성'임.
    - margin: 10px 20px 30px 40px; 로 정의 가능.

## box-sizing:

    - 요소의 너비와 높이를 계산하는 방법을 지정.
    - content-box: 기본값으로 너비와 높이가 콘텐츠 영역만을 포함.
    - border-box: 너비와 높이가 안쪽 여백과 테두리까지 포함.

## background:

    - 콘텐츠의 배경을 정의함.
    - background-color: 배경 색 정의
    - background-image: 배경 이미지 정의
    - background-position: 배경 이미지의 초기 위치 정의
    - background-size: 배경 이미지 크기 정의
    - background-repeat: 배경 이미지 반복 방법 정의

## float:

    - 요소가 문서의 (먼저 작성된 것 부터 브라우저에 뜨는) 일반적인 흐름에서 제외되어 자신을 포함하고 있는 컨테이너의 왼쪽이나 오른쪽에 배치되게 함.
    - none: 기본값. 원래 상태.
    - left: 자신을 포함하고 있는 박스의 왼편에 떠 있어야 함.
    - right: 자신을 포함하고 있는 박스의 오른편에 떠 있어야 함.

## clear:

    - float 요소 이후에 표시되는 요소가 float을 해제(clear)하여 float 요소의 아래로 내려가게 할 수 있음
    - none: 기본값으로 아래로 이동되지 않음을 나타내는 키워드
    - left: float이 left인 요소의 아래로 내려가겠다.
    - right: float이 right인 요소의 아래로 내려가겠다.
    - both: float이 right 및 left인 요소의 아래로 내려가겠다.

## position:

    - 문서 상에 요소를 배치하는 방법을 정의.
    - position이 요소가 배치될 방법을 결정하면 top, bottom, right, left가 얼만큼 떨어뜨릴건지를 결정하여 최종 위치를 정함.

## position 속성값:

    - 오프셋(보충): 위치를 얼마간 이동시키는 것
    - static: 기본값. 요소를 일반적인 문서 흐름에 따라 배치.
    - relative: 일반적인 문서 흐름에 따라 배치하되, 상하좌우 위치 값에 따라 오프셋 적용
    - absolute: 일반적인 문서 흐름에서 제거하고 가장 가까운 position 속성에 속성값이 정의되어 있는 요소에 대해 상대적으로 오프셋 적용.
    - fixed: 일반적인 문서 흐름에서 제거하고 지정 위치에 고정됨.
    - sticky: 일반적인 문서 흐름에서 제거하고, 스크롤 동작이 존재하는 가장 가까운 요소에 대해 오프셋 적용. 스크롤 이동으로 요소가 움직여도 스티키 요소는 고정된 상태를 유지.

## 특성 선택자 1. 컨셉[]:

    - 주어진 속성의 존재 여부나 그 값에 따라 요소를 선택.
    - [class]{~~~} : 클래스 속성을 가지고 있는 요쇼를 선택
    - [class="item"]{~~~}: 클래스가 "item"인 요소를 선택.

## 특성 선택자 2. 값 확인:

    - 기호를 추가해 요소 선택 방식을 다양화 가능.
    - [class *= "it"]: 클래스 값에 "it"가 포함되는 요소 선택
    - [class ^= "it"]: 클래스 값이 "it"로 시작하는 요소 선택
    - [class $= "it"]: 클래스 값이 "it"가 끝나는 요소 선택

## 결합 선택자 1. 컨셉:

    - 두 개 이상의 선택자를 결합시켜 결합된 조건을 만족하는 요소를 선택.

## 결합 선택자 2. 자손 결합자:

    - 두 개의 선택자 중 첫 번째 선택자 요소의 자손을 선택 가능.
    - div p {color:white;}: div 요소 안에 있는 모든 p 요소 선택
    - div > p {color:white;}: div 요소 바로 아래에 위치하는 모든 p 요소 선택.

## 결합 선택자 3. 형제 결합자:

    - 두 개의 선택자 중 첫 번째 선택자 요소의 형제를 선택 가능.
    - h1 ~ p{color:red;}: h1 요소의 뒤에 오는 형제 중 모든 p 요소를 선택
    - h1 + p{color:red;}: h1 요소의 바로 뒤에 오는 형제 p 요소를 선택

## 의사클래스:

    - 선택자에 추가하는 키워드로, 요소가 어떤 특정한 상태가 되었을 때 요소를 선택하겠다는 의미.
    - hover: 마우스 포인터가 요소에 올라가 있음.
    - active: 마우스 누르기와 같이 요소 활성화됨
    - focus: 포커스 받고 있음
    - disabled: 비활성 상태
    - nth-child(): 형제 사이에서의 순서에 따라 요소 선택

## 의사요소:

    - 선택자에 추가하는 키워드로, 이를 이용하면 선택한 요소의 특정 부분에 대한 스타일 정의 가능.
    - 사용예시) 선택자::의사요소{속성명: 속성값;}
    - after: 요소의 앞에 의사 요소를 생성 및 추가
    - before: 요소의 뒤에 의사 요소를 생성 및 추가
    - first-line: 블록 레벨 요소의 첫 번째 선에 스타일 적용
    - marker: 목록 기호의 스타일 적용
    - placeholder: 자리표시자 스타일 적용
    - HTML 문서가 포함하지 않는 개체(목록 구분점, 플레이스홀더 등)를 선택할 수 있어 유용함.

## 상속:

    - 하위 요소가 상위 요소의 스타일 속성값을 물려받는 것.
    - 다 되는 것은 아님.
    - 상속 O: color, font-family, font-size, cursor 등
    - 상속 X: background 관련, border, display 등

## 공용 키워드:

    - 모든 CSS 속성에 사용 가능한 키워드. 때문에 "전역 값"이라 표현하기도.
    - inherit: 상위 요소로부터 해당 속성의 값을 받아 사용.
    - initial: 브라우저 지정 해당 속성의 기본값을 요소에 적용.
    - unset: 상속 속성에 대해선 inherit처럼, 상속되지 않는 속성에 대해선 initial처럼 적용.

## z-index:

    - 요소의 쌓임 순서(stack order)를 정의 가능.
    - 정수 값을 지정하여 쌓임 맥락에서의 레벨을 정의하는 방식. 값이 클수록 전면에 표시됨.
    - 위치 지정 요소(position 속성이 정의되어 있는 요소)에 대해 적용 가능한 속성.
    - 쌓임 맥락: 동일한 위치에 요소들이 배치되면 요소들은 z축에 쌓이게 됨.
    - 기본값: auto. 새로운 쌓임 맥락이 형성되지 않은 상태임.
