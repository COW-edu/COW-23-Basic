## HTML:

    - HTML(HyperText Markup Language)은 웹 페이지의 구조 혹은 데이터 작성을 위한 마크업 언어이다.
      HTML5 문서는 반드시 <!DOCTYPE html>으로 시작하며 이는 문서 형식을 HTML5로 지정한다는 의미이다.
      <head>와 </head> 사이에는 문서의 제목, 외부 파일의 참조, 메타데이터의 설정 등이 위치하며 이 정보들은
      브라우저에 표시되지 않는다.

## HyperText란 무엇인가?:

    - 하이퍼 링크를 이용해 독자가 원하는 순서로 기존 문서에서 다른 문서로 접근할 수 있는 텍스트이다.
      즉, 책처럼 한 장 한 장 차례대로 내용을 읽는 전통적인 텍스트 접근 방식이 아닌, 링크라는 통로를 통해
      원하는 페이지로 바로 도착하는 접근 방식을 가지고 있다.
      이렇게 하이퍼 텍스트는 서로 다른 텍스트들이 다중으로 연결되어 있어, 정보에 꼭 순차적으로 접근할 필요 없이
      바로 사용자가 원하는 정보에 접근할 수 있다.

## Markup Language이란 무엇인가?:

    - 마크업(Mark Up)이란 어떠한 강조하는 표시를 뜻하며, 마크업 언어는 태그를 이용하여 문서나 데이터의 구조를 명시하는
      언어를 말한다. 표시를 하여 문서를 구조화한다는 말이 와닿지 않을 수 있지만, 이는 우리 주변에서 꽤 흔히 쓰이는
      방법이다. 예를 들어, 택배를 보내기 위해 정보를 기록했는데 어떤 정보가 더 보기에 편할까.

      1. 경기도 성남시 분당구 판교로 242 판교디지털센터 A동, 구름(goorm), 031-xxx-xxxx
      2.주소: 경기도 성남시 분당구 판교로 242 판교디지털센터 A동
        수령인: 구름(goorm)
        연락처: 031-xxx-xxxx

      2번 방식처럼 각 정보가 어떠한 의미를 가지는지 표시한 것을 더 선호할 것이다.
      여기서 주소, 수령인, 연락처 같은 표시가 바로 마크업에 해당한다.

      <제목> 바로 실행해보면서 배우는 HTML/CSS </제목>
      <부제목> 구름과 함께하는 쉽고 빠르게 배우는 HTML과 CSS </부제목>
      <내용> HTML과 CSS의 전반적인 설명을 담은 내용 </내용>

      홑화살괄호인 '<>' 안에 각 텍스트가 무엇을 의미하는지 '마크업' 되어 있다.
      이처럼 표시가 어떤 정보를 의미하는지 파악할 수 있다.

## 메타데이터란?:

    - <meta> 태그를 이용해 표현하며 화면에 표시되는 내용이 아닌 HTML 문서 자체에 대한 데이터라고 할 수 있다.
      예를 들어 <meta charset="utf-8">은 문서의 문자셋 정보를 추가해 다양한 문자를 인코딩할 수 있도록 설정한 부분이다.
      그러면 메타데이터를 추가하는 이유는 무엇일까? 웹 페이지의 서비스 품질을 향상시키기 위함이다.
      <meta>태그를 이용하면 문자 인코딩 방법을 비롯해 웹 문서 키워드 등을 지정할 수 있는데,
      이러한 정보들은 검색 엔진이 웹 페이지를 검색하는 데 유용한 정보로 활용된다.

## 표준 메타데이터 사용 예시:

    - 문서 제작자 정보를 나타내는 author. (문서를 작성한 사람에 대한 정보를 추가함)
      >  <meta name="author" content="aumjueun">
    - 웹 페이지에 대한 짧고 정확한 요약을 추가하는 description.
      (파이어폭스, 오페라 등 여러 브라우저에선 즐겨찾기 페이지의 기본 설명 값으로 description의 값을 사용하기도 함)
      >  <meta name="description" content="엄주은의 웹 공부 기록을 담고 있는 블로그입니다.">
    - 웹 페이지의 콘텐츠와 관련된 키워드를 추가하는 keywords.
       >   <meta name="keywords" content="animal, cat, dog, puppy, kitty">
    - 모바일 기기에서의 사용을 위한 viewport.
      (viewport란 기기의 화면 상에서 콘텐츠가 표시되는 영역을 뜻하는 용어.
      PC 화면과 모바일 화면은 뷰포트 크기 자체가 다르고, 또 그 안에서 화소(픽셀)를 표현하는 규모 또한 다르기 때문에
      PC 화면과 모바일에서는 같은 코드의 페이지라도 배율 조정 때문에 다르게 보이는 현상이 발생하곤 함.
      메타데이터 viewport는 그 값을 어떻게 설정하는가에 따라 이러한 문제를 해결할 수 있는 메타데이터이다.
      아래 코드는 viewport 지정 시 가장 많이 사용하는 형태이다.)
       >  <meta name="viewport" content="width=device-width, initial-scale=1.0">

## 태그:

    - HTML 코드에서 정보(콘텐츠)를 정의하는 형식

## 태그 사용 형태:

    - > <태그명></태그명>
    - 시작과 끝을 구분치 않는 단일 태그도 존재.

## 속성:

    - 태그의 부가적인 기능 정의
    - 시작 태그의 내부에 정의
    - 개수 제한 없음
    - > 태그명 속성명 = "속성값"></태그명>

## &#60;!DOCTYPE html>:

    - 문서의 첫 부분에서 문서 유형을 지정하는 단일 태그

## 줄바꿈 태그와 공백문자:

    - <br> : 줄바꿈
    - &nbsp; : 공백문자(no-break space)
    - 스페이스는 한 번씩만 허용된다.
    (연달아 입력할 경우 한 번 외에는 무시한다.)

## 헤드라인:

## 블록 레벨 요소:

    - 블럭요소 : <div>,<table>,<h1>~<h6>,<p>,<form>,<ul>,<ol>,<li>,<dl>,<dt>,<dd>,<pre>,<blockquote>등

## 인라인 요소:

    - 자기에게 필요한 만큼의 공간만을 차지함
    - 인라인요소 : <span>,<a>,<br>,<em>,<strong>,<input>,<label>,<img>

## url:

    - 이미지가 어디에 있는지 알려주기 위해 사용하는 규약
    - URL(Uniform Resource Locator)은 네트워크 상에서 자원이 어디 있는지를 알려주기 위한 규약이다.
      흔히 웹 사이트 주소로 알고 있지만, URL은 웹 사이트 주소뿐만 아니라 컴퓨터 네트워크 상의 자원을 모두 나타낼 수 있다.
      그 주소에 접속하려면 해당 URL에 맞는 프로토콜을 알아야 하고, 그와 동일한 프로토콜로 접속해야 한다.

## URL의 구조:

    - 통신방법:[//[사용자ID:비밀번호@]주소[:포트번호]][/]경로[?추가정보]

## alt(이미지 설명):

    - alternative의 약자로 대체 텍스트 역할
    - 이미지 로딩 전이나 로딩 실패의 경우 이미지 대신에 대체 텍스트가 표시됨
    - 음성인식기가 있어 시각장애인에게 서비스 가능

## 컨테이너 태그:

    - 콘텐츠나 레이아웃에 아무런 영향도 주지 않으며 단지 다른 요소 여럿을 묶어 관리하기 편하게 만드는 역할을 하는 태그
    - 콘텐츠 내용을 구분하거나, 공통적인 스타일을 적용하고자 할 때 사용하는 것이 좋음

      > div(블록)와 span(인라인)

## 전역 속성:

    - 모든 태그에 공통적으로 지정 가능한 속성

      accesskey: 요소 선택/지정 단축키 지정
      class: 요소 참조를 위한 클래스 지정
      contenteditable: 요소 내용 수정 유/무 지정
      data-\*: 요소에 사용자 속성 지정
      dir: 요소 문자 흐름 지정
      draggable: 요소 드래그 지정
      hidden: 요소 숨김 지정
      id: 요소 고유 아이디 지정
      lang: 문서 내용에 사용된 언어 지정
      spellcheck: 요소 내용의 철자, 문법 체크 여부 지정
      style: 요소에 스타일 지정
      tabindex: 요소에 탭 순위 지정
      title: 요소에 추가 정보 지정
      translate: 요소의 번역 여부 지정(지원되는 브라우저 없음)

## 링크는 anchor:

    - a 태그 요소는 href 속성을 통해 다른 url로 연결할 수 있는 링크를 만듬(인라인 요소임)
    - 현재 탭에서 열기(기본값)
      > target="\_self"
    - 새 탭에서 열기
      > target="\_blank"
    - 현재 프레임의 부모 프레임에서 새 웹페이지가 열림. 만약 어떤 창 A에서 창 B를 새로 열었다.
      그런데 창 B에서 \_parent로 창을 열면 창 A에서 새 웹페이지가 열리게 된다.
      > target="\_parent"
    - 최상위 프레임에서 열림. \_parent와 비슷하지만 얘는 바로 이전창(부모창)에서 열리는 반면
      \_top은 최상위(가장 최고 부모)창에서 열림.
      > target="\_top"
